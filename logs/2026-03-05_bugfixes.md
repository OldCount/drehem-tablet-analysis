## drehem_extract.py

### @t shape notation
- Verified already handled: NUMERAL_PATTERN includes disz@t variant and
  NUMERAL_MULTIPLIERS maps it to 1. No change needed.

### Day parser improvements
- Extended day regex to also match `ba-ra-zal` as a terminator alongside
  `ba-zal` and `-kam`, so patterns like `u4 N ba-ra-zal` are parsed correctly.
- Added `iti-ta u4 N ba-ra-zal` formula: "of the month, day N has passed".
  Attested in P123620. `la2` subtraction already handled by `parse_numeral`.

### Komma notation normalisation
- Added `s, → s` substitution in `strip_atf_damage()`. The comma is an ATF
  convention for the sibilant ṣ and was causing name matching failures for
  names like `s,e-lu-usz-{d}da-gan` (Seluš-dagan) in P128123.

### City name recognition
- Added `nibru{ki}` (Nippur), `uri2{ki}` (Ur), `lagasz{ki}`, `umma{ki}`,
  `irisagrig{ki}`, `hu-ur5-ti{ki}` to DESTINATION_CATEGORIES as `city` type.
  These appear in `ensi2 CITY{ki}` lines and `giri3` intermediary contexts.

## tablet_visualizer.py

### Damage token fix
- Fixed indentation bug: the `if role == "unknown"` fallback was nested inside
  the `elif token is damage` branch, making it unreachable. Moved it to the
  correct level so all tokens without a matched role fall back to `text`.
- Added explicit `[...]` to the damage token pattern.

### Transaction keyword matching — ba-ba-ti maszkim fix
- Replaced the loose `content.startswith(k)` check with a proper token-by-token
  exact match using two new helpers:
    - `_matches_transaction_keyword(token, tokens, idx)`: returns True only when
      the token and its successors exactly match the keyword's parts
    - `_get_transaction_keyword(token, tokens, idx)`: returns the matched keyword
- This prevents `ba-ti` inside `ba-ba-ti` from being falsely tagged as part of
  the two-token keyword `szu ba-ti` (receipt).
- Multi-token keywords now also advance the token index correctly, preventing the
  following token (`ba-ti`) from being re-processed independently.

### Divine recipients panel
- Rendering updated to use `divine_recipients_translated` when available.
  Each deity is shown with its Sumerian name and English/Dutch translation below.
  Language responds to the EN/NL toggle button. Falls back to raw name list if
  translation data is absent.

# Worklog: 2026-02-28

## 1. Fix: Dashboard HTTP Handler Serving Path

### Problem
`DashboardHandler` in `run_dashboard.py` subclassed `BaseHTTPRequestHandler` and only served its inline HTML template. It had no static file serving capability, so when the server was started from a directory other than `cdli_data/`, requests for `dashboard.html`, `diagnostics_data.json`, and `oracc_name_dictionary.json` would fail.

### Fix
Switched `DashboardHandler` to subclass `SimpleHTTPRequestHandler` with `directory=str(BASE_DIR)` in `__init__`, matching the pattern already used by `VisualizerHandler` in `tablet_visualizer.py`. The handler now:
- Serves the inline HTML template at `/` (root path)
- Handles the `/rerun` API endpoint
- Falls through to `SimpleHTTPRequestHandler.do_GET()` for all other paths, serving files from the script's parent directory regardless of working directory

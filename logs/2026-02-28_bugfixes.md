# Bug Fixes & Edge Total Overhaul: 2026-02-28

## Diagnostics Comparison (2000-tablet sample)

| Metric | Before | After |
|--------|--------|-------|
| Errors | 36 | 20 |
| Warnings | 92 | 85 |
| total_mismatch | 84 | 47 |
| unrecognized_after_numeral | 97 | 99 |

## 1. Fixed `assess_damage()` broken_signs accumulation bug

Line 511 had `report.total_signs += report.broken_signs` inside the per-line loop.
Since `broken_signs` is a running total across all lines, each iteration added the
full cumulative count again, grossly inflating `total_signs` and deflating
preservation scores.

Fix: track per-line bracket tokens in a local `line_broken` variable.

## 2. Overhauled `_extract_edge_total()` to use `@left` ATF sections

Previously: scanned last 5 content lines for standalone numerals. This grabbed
date numerals, animal counts, and other content as false edge totals.

Now: parses the `@left` structural section from raw ATF (2,278 tablets have one,
1,436 with a parseable numeral total). Tablets without `@left` return 0 rather
than guessing.

Result: total_mismatch errors dropped 44% (84 → 47). Remaining mismatches are
genuine data quality issues (damaged tablets, unusual scribal notation).

## 3. Excluded `@left` from `extract_content_lines()`

Left-edge content (typically the scribe's running total) was being fed into the
extraction pipeline as if it were tablet content. Now `@left` sections are
skipped alongside `@seal`, preventing double-counting.

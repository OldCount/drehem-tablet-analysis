# Worklog: 2026-02-26

## Overview
This document records the modifications made to the Drehem extraction pipeline, diagnostic tools, and user interface on 2026-02-26. The main objectives were to automate the workflow, improve the algorithm's accuracy specifically regarding ambiguous counts and measure terms, and unify the diagnostic tools into a single coherent interface.

## 1. Workflow Automation

### `drehem.sh` (New File)
Developed a comprehensive shell script to manage the extraction pipeline.
- Implemented freshness checks utilizing `is_fresh` bash function to compare output modification times against source files, preventing unnecessary rebuilds of the dictionary or the main extraction dataset unless `--force` is provided.
- Added a `setup` command to install a global `drehem` alias within the user's `~/.zshrc`, allowing execution from any directory.
- Created distinct subcommands:
  - `status`: Displays the current freshness state of pipeline outputs.
  - `dict`: Rebuilds the name dictionary (`build_name_dictionary.py`).
  - `extract`: Runs the main extraction script (`drehem_extract.py`).
  - `search` / `dashboard`: Serves the respective HTML interfaces on designated ports (8090 and 8787).
  - `all`: Sequentially runs applicable stale steps and automatically opens the unified studio view.
  - `backup`: Creates a timestamped directory within `backups/` containing all current scripts and HTML files, excluding large CSVs and cache directories.

### Launcher Shortcuts
- Created `Open Name Search.command` and `Open Dashboard.command`. These double-clickable scripts execute the corresponding `drehem.sh` commands, providing immediate GUI access without manual terminal execution.

## 2. Algorithm Refinements (`drehem_extract.py`)

Seven specific algorithmic issues were addressed in the main extraction logic:

1.  **Measure Terms:** Integrated a `MEASURE_TERMS` constant (comprising terms like `barig`, `ban2`, `sila3`, `ga-sze-a`) to prevent false-positive "unrecognized after numeral" warnings during diagnostic runs.
2.  **Damaged Summary Lines:** Modified diagnostic scripts (`build_diagnostics.py` and `run_dashboard.py`) to suppress the `total_mismatch` warning when the summary line itself contains damage markers (`[...]`, `[x]`, `#`), acknowledging structural uncertainty.
3.  **Leading Dot Lines:** Updated `extract_content_lines` to correctly parse lines beginning with `. ` (a standard ATF continuation marker). This successfully resolved the issue where intercalary month dates (e.g., `iti diri ezem-{d}me-ki-gal2 ba-zal`) were silently ignored.
4.  **Destination Categorization:** Added `du6-ku3` to `DESTINATION_CATEGORIES` with the classification `"sacred-site"`.
5.  **Bala System Officials:** Introduced the `BALA_PATTERN` regex to specifically handle the `bala PN ensi2 CITY` structure. The function `extract_persons` was updated to classify these individuals as `source` with a distinct `name_type="bala-official"`, simultaneously flagging the `source_office`.
6.  **Theophoric False Positives:** Suppressed the `possible_theophoric` warning in diagnostic scripts when the identified deity string is already incorporated within a successfully extracted personal name (resolving false alarms like `puzur4-{d}en-lil2`).
7.  **Uncertain Counts (`[x]`):** Restructured `extract_animals` to perform a secondary pass matching `^\[x\]\s+` on the **raw** line (prior to `strip_atf_damage` execution, which inadvertently removed bracket context). These entries are now recorded with `count=1` and `damaged=True`, ensuring they exclusively contribute to `total_animals_uncertain` to maintain academic transparency.

## 3. Unified Studio Interface (`dashboard.html`)

Rebuilt the diagnostics dashboard into a comprehensive, three-panel "Studio" view, merging the previously separate diagnostic list and name dictionary tools.

- **Layout:** Implemented a responsive grid (tablet list, tablet detail, dictionary sidebar).
- **Data Integration:** The page now fetches both `diagnostics_data.json` and `oracc_name_dictionary.json` from the same local server instance.
- **Cross-linking:** Added interactivity to the tablet detail panel. Clicking any extracted personal name or divine recipient automatically triggers a search within the dictionary sidebar panel.
- **Default Sample Size:** Bumped the default sample size from 500 to 1000 in both `build_diagnostics.py` and `run_dashboard.py` to provide a broader analytical overview.
